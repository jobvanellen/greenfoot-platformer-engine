# Greenfoot Platformer engine
In deze git repository vind je de basiscode van je greenfoot project. Dit project kan je openen met IntelliJ. 
Nadat je dit project hebt geopend moet je de App.java klasse instellen als main zodat je het project kan starten.

In deze repo vind je de package `demo` en de package `engine`, deze mogen jullie niet aanpassen voor het geval wij updates 
willen uitbrengen.

Om je project te starten kan je een nieuwe package maken genaamd `game` waarin je jullie project gaat starten.

In de `demo` package hebben wij een klein voorbeeld gemaakt van hoe je een project start. Dit kan je gebruiken of namaken.

## Samenwerking en Git Flow
Om goed samen te kunnen werken is het verstandig om volgens een bepaalde git structuur te werken. 
Dit wordt ook wel een git flow genoemd. Er zijn verschillende vormen van git flows. Wij gaan werken met de volgende 
git flow:

* master
* development
* feature-[naam feature]

Op de master branch wordt niet gewerkt en op de development branch ook niet. Dit zijn branches waar versies op gemaakt worden.
Je werkt alleen in een feature branch. Zodra een feature af is wordt deze gemerged met de development branch en daar kan
het team die feature testen en er feedback op geven. Zodra alle features gemaakt zijn voor een sprint en getest zijn op 
de development branch, dan wordt er een merge gemaakt met de master (en dus een nieuwe versie gemaakt).
